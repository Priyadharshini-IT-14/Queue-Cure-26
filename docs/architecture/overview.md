# QueueCure Pro Architecture

QueueCure Pro uses a plain HTML, CSS, and JavaScript frontend with a Node.js and Express.js backend.

The frontend is organized by pages, components, layouts, services, context, routes, hooks, and utilities. The backend is organized by controllers, routes, middleware, models, services, sockets, config, utilities, and seed data.

Current data is stored in memory for the demo. Tokens start empty and are generated in order: `A001`, `A002`, `A003`, and so on.
